01004-5:Greedy algorithms

Posted on 2020-11-13 Edited on 2020-12-05 In 01004

Greedy algorithms

Fractional Knapsack

  • [highest benefit]: Select items with highest benefit.
  • [smallest weight]: Select items with smallest weight.
  • [benefit/weight]: Select items with highest benefit to weight ratio.

The greedy strategy of picking item with highest benefit to weight ratio computes an optimal solution.

Require O(n log n) time to sort the items and then O(n) time to process them in the for- loop.

Interval Scheduling

  • [Earliest start time]
    Consider jobs in ascending order of start time sj.
  • [Shortest interval]
    Consider jobs in ascending order of interval length fj - sj.
  • [Fewest conflicts]
    For each job, count the number of conflicting jobs cj. Schedule in ascending
    order of conflicts cj.
  • [Earliest finish time]
    Consider jobs in ascending order of finish time fj.
    O(n log n).

Greedy algorithm [Earliest finish time] is optimal.

Interval Partitioning

Greedy algorithm never schedules two incompatible lectures in the same classroom.

O(n log n).

Shortest Paths in a Graph

Dijkstra’s Algorithm

Using a heap for PQ, Dijkstra runs in O(m log n) time
Using a Fibonacci heap for PQ, Dijkstra runs in O(m + n log n) time

Minimum Spanning Tree

Greedy Algorithms to find MST

  • Kruskal’s algorithm
    Start with T =$\emptyset$ Consider edges in ascending order of cost. Insert edge e in T unless doing so would create a cycle.
    using a union-find data structure it takes O(m log n) time

  • Reverse-Delete algorithm
    Start with T = E. Consider edges in descending order of cost. Delete edge e from T unless doing so would disconnect T.

  • Prim’s algorithm
    Start with some root node s and greedily grow a tree T from s outward. At each step, add the cheapest edge e to T that has exactly one endpoint in T.
    O(m log n) with a heap; O(m + n log n) with a Fibonacci heap.

Finding cut edges in O(n+m) time

Testing Bipartiteness

An undirected graph G = (V, E) is bipartite if the nodes can be colored red or blue such that every edge has one red and one blue end.
If a graph G is bipartite, it cannot contain an odd length cycle.

Connectivity in Directed Graphs

Directed Graphs

Strong Connectivity

A graph is strongly connected if every pair of nodes is mutually reachable.
Can determine if G is strongly connected in O(m + n) time.

DAGs and Topological Ordering

Directed Acyclic Graphs (DAGs)

A DAG is a directed graph that contains no directed cycles.

Topological Ordering

A topological order of a directed graph G = (V, E) is an ordering of its nodes as v1, v2, …, vn so that for every edge (vi, vj) we have i < j.

Algorithm finds a topological order in O(n + m) time.

If G has a topological order, then G is a DAG.
If G is a DAG, then G has a node with no incoming edges.
