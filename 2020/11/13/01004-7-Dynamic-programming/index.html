Posted on 2020-11-13 Edited on 2020-12-05 In 01004 Views:

Dynamic programming

  1. Define subproblems
  2. Find recurrences
  3. Solve the base cases
  4. Transform recurrence into an efficient algorithm

Weighted Interval Scheduling

  1. Define subproblems (DP states)
    OPT(j) = value of optimal solution to the problem consisting of job requests 1, 2, …, j.

  2. Find recurrences

  • Case 1: OPT selects job j.
  • Case 2: OPT does not select job j.
    OPT(j) = max {vj + OPT (p(j)), OPT(j-1)}
  1. Solve the base cases
    OPT(0) = 0

O(n) if jobs are pre-sorted by start and finish times.

Maximum-sum contiguous subarray

By Divide-and-conquer

T(n) = 2·T(n/2) + O(n) = O(n log n)

By Dynamic programming

MaxHere[i] - optimal solution ending at i
MaxHere[i] = max{MaxHere[i-1]+A[i],0}
Total time: O(n)

Knapsack Problem

OPT(i) = max profit subset of items 1, …, i.

  • Case 1: OPT does not select item i.
  • Case 2: OPT selects item i.
    OPT(i,w) = max { vi + OPT (i-1,w-wi), OPT(i-1,w) }

Running time:

Longest Increasing Subsequence

OPT(i,j) = length of LCS(X[1..i], Y[1..j]).

Running time: O(nm)
Space: O(nm)

Shortest Paths

Attempts by Dijkstra

Negative edge costs. FAILED

OPT(i, v) = length of shortest v-t path P using at most i edges.
Case 1: P uses at most i-1 edges.
OPT(i, v) = OPT(i-1, v)

Case 2: P uses exactly i edges.
if (v, w) is first edge, then OPT uses (v, w), and then selects best w-t path using at most i-1 edges

$OPT(i,v) =min{OPT(i-1,v), min [OPT(i-1,w)+c_{vw}] } (v,w)\in E$
