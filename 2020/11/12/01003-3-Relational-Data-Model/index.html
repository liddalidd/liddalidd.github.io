Relational Data Model

Logical Database Design

Relational model (relation and schema)

  • relation schema: name of relation, and name and data type (domain) of each attribute.
  • relation instance: set of tuples (a table) for a schema
  • NULL:
    • pro: using an ordinary value with special meaning does not always work
    • con: causes complications in the definition of many operations

Data definition language (DDL)

Integrity constraints

facilities to specify a variety of rules to maintain the integrity of data when it is manipulated

  • Key: the minimal set of attributes in a relation that can uniquely identify each row of that relation

  • Foreign keys (FK) are identifiers that enable a dependent relation to refer to its parent relation

  • Primary Key= Not Null + Unique

Mapping E-R diagrams to relations

  • N:N Relationship:
    E1-R-E2 -> E1+(E1_PK,E2_PK)+E2
  • N:1/1:N Relationship:
    E1-R-En -> E1+ En_E1_PK
  • 1:1 Relationship:
    E1-R-E2 -> E1+ E2_E1_PK / E2+ E1_E2_PK
  • Weak Entity Types:
    S-R-W -> S + W_S_PK
  • IsA-Hierarchy:
    Super-IsA-Sub -> Super + Sub_Super_PK
  • Aggregation:
    A-R-E -> A + (A_PK,E_PK) + E
