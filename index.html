YARN

MapReduce 1.0

缺陷：单点故障、JobTracker 任务过重、容易内存溢出、资源化分不合理

设计思路

ResourceManager: 资源管理
ApplicationManager: 任务调度、任务监控
NodeManager: TaskTracker

体系结构

ResourceManager：处理客户端请求、启动/监控ApplicationManager、监控NodeManager、资源分配与调度
ApplicationManager：为应用程序申请资源，分配任务、任务调度、监控与容错
NodeManager: 单个节点的资源管理、处理命令

工作流程

  1. 向yarn提交应用程序；
  2. ResourceManager接受处理、分配容器并启动ApplicationManager；
  3. ApplicationManager创建后向ResourceManager注册；
  4. ApplicationManager轮询 向ResourceManager申请资源；
  5. ResourceManager以容器的形式对ApplicationManager分配资源；
  6. 在容器中启动任务（分配Map/Reduce 任务）；
  7. 各个任务向ApplicationManager汇报状态和进度；
  8. 运行完成后，ApplicationManager向ResourceManager的应用程序管理器注销并关闭自己。

Evaluating Clustering

Cluster Validity

Measures of Cluster Validity

  • External Index: Measure the extent to which cluster labels match externally supplied class labels (e.g., accuracy, precision, recall, F1-score)

    • Homogeneity: precision P = TP / (TP+FP)
    • Completeness: recall R = TP / (TP+FN)
    • V-measure: F1 score = 2PR / (P+R)

  • Internal Index: Measure the goodness of a clustering structure without respect to external information (e.g., Sum of Squared Error)

    • $SSE=\sum_{i=1}^K\sum_{x\in C_i}dist^2(m_i,x)$

    • Silhouette Coefficient:
      $s=1-a/b if a<b else s=b/a-1$

  • Relative Index: Compare two different clusterings or clusters (often an external or internal index is used)
